# Cover Letter & Interview Templates (for Applicants)

> Reference file. Load only when the user needs a cover letter, application message, interview follow-up, or offer negotiation script. Context pointer: *Resume & Cover Letter branch / Interview Prep branch / Job Search Strategy & Offer Negotiation branch: see reference for full templates.*

## Template Types

### 1. Cold Application Cover Letter (Email or Portal)

Use when applying via a company's careers page, email, or direct submission. The entire letter leans on **shipped code**: the resume is attached; the cover letter sells the proof.

```
## Application: [Role] at [Company]

**To:** [Hiring Manager Name] or [Team]

### Hook (1 sentence)
Lead with what you've shipped that's relevant to them.

I'm applying for the [Role] role at [Company]. I've shipped [project] at
github.com/armchairfuturist-code/[repo]: a [1-sentence description] that
handles [key capability] using [tech stack].

### GitHub Proof (the core)
You can inspect the architecture, commit history, and README at the repo above.
The same level of craft, clean code, documented decisions, shipped iteration:

### Bridge
Why [Company]? [1-2 sentences specific to their product, blog post, or
engineering challenge: demonstrate you've done your homework.]

### Close
I'd love 15 minutes to walk your team through [project] and talk about how I'd
approach [their specific challenge]. My GitHub is my resume: you can look at
the commit history before we talk.

Best,
Alex Myers
github.com/armchairfuturist-code
```

**Key rule:** Never send a generic cover letter. The GitHub project must be **relevant to what the company does**. If you don't have a directly relevant repo, describe a public project that exercised analogous skills and say "same architecture, different domain."

### 2. Warm Intro Message (DM or Email via Referral)

Use when someone at the company (or a mutual connection) agrees to make an intro. Shorter, warmer, still GitHub-first.

```
**Subject:** [Referral Name] suggested I reach out: shipped [project]

Hi [Name],

[Referral Name] suggested I connect. I've shipped [project] at
github.com/armchairfuturist-code/[repo]: a [description] that [capability].
I'm exploring full-time roles at companies building [industry/sector], and
[Company]'s work on [product] caught my eye.

Would you have 10 minutes this week to chat? I'd love to hear about what
[Company] is building and share how I've approached similar problems.

Best,
Alex Myers
github.com/armchairfuturist-code
```

---

### 3. LinkedIn InMail / DM (Cold to Recruiter or Hiring Manager)

Use when direct-messaging a recruiter, hiring manager, or tech lead on LinkedIn. Short, high-signal, one-click to GitHub.

```
Hi [Name],

I ship AI systems: 40+ of them, code at
github.com/armchairfuturist-code. I'm exploring full-time roles at
early-stage startups and [Company]'s [product/team] looks like a fit.

Would you have a few minutes to see if there's a match?

Alex
```

**Note:** This is a bolt: short enough to read on mobile. The GitHub link does the heavy lifting.

---

### 4. Follow-Up After Application (No Response)

Send 7-10 days after applying if no response. Keep it short. Don't re-pitch. Just remind them where your GitHub is.

```
**Subject:** Follow-up: [Role] applicant: [Project] at github.com/armchairfuturist-code

Hi [Name],

Following up on my application for the [Role] role. I wanted to make it easy
to see the kind of work I ship: github.com/armchairfuturist-code/[repo].

If the role is still open, I'd love 15 minutes to walk your team through the
architecture and talk about how I'd contribute to [Company].

Best,
Alex Myers
github.com/armchairfuturist-code
```

### 5. Thank-You After Interview

Send within 4 hours of the interview. Reference the conversation.

```
**Subject:** Thanks: [Role] at [Company]

Hi [Name],

Thanks for the time today. I enjoyed our conversation about [specific topic
discussed: e.g., their architecture challenge, the product roadmap, the team
structure].

github.com/armchairfuturist-code/[repo]: the commit history shows how I

I'm excited about the possibility of contributing to [Company]. Happy to
answer any follow-up questions or do a deeper walkthrough.

Best,
Alex Myers
github.com/armchairfuturist-code
```

### 6. Offer Evaluation & Negotiation Script

Use when you have an offer in hand and need to evaluate or negotiate.

```
## Offer Evaluation: [Company]

### The Offer
- Role: [Title]
- Base: [$X]
- Equity: [X% / X options at $Y strike]
- Other: [Signing bonus, benefits, etc.]

### Evaluation
| Factor | Good | Concern |
|---|---|---|
| Base | [market for stage?] | [below market?] |
| Equity | [stage-appropriate %?] | [below expected?] |
| Role scope | [aligned with growth?] | [narrower than discussed?] |
| Team/Manager | [strong signal?] | [unknown?] |
| Product fit | [can use your skills day 1?] | [pivot risk?] |
| Flexibility | [remote, hours, start date] | [rigid?] |

### Negotiation Script (if needed)

**Option 1: Base salary**
> "I'm excited about the role and the team. Based on the scope and my
> experience shipping [relevant systems], I was hoping for something closer
> to [$X + Y%]. Is there flexibility on base?"

**Option 2: Base is fixed, negotiate equity/title/flexibility**
> "I understand the base is firm. Could we look at equity or a 6-month
> performance review with a defined increase?"

**Option 3: Multiple offers (only if true)**
> "I have another offer on the table, but [Company] is my first choice. If
> you can match [$X], I'll accept today."
```

### 7. Take-Home Project README Template

If the company sends a take-home assignment, use this as the README structure:

```
# [Project Name]: Take-Home for [Company]

## What it does
[1-2 sentences]

## Architecture
[Key decisions and trade-offs. What did you choose and why?]

## How to run
[Clear instructions]

## What I'd add with more time
[Shows you think beyond the spec: optional, strong signal]
```

## Quick Reference: Which Template for Which Situation

| Situation | Use Template |
|---|---|
| Applying via careers page / email | #1: Cold Application Cover Letter |
| Warm intro from a connection | #2: Warm Intro Message |
| Cold DM on LinkedIn | #3: LinkedIn InMail |
| No response after 7-10 days | #4: Follow-Up After Application |
| Just finished an interview | #5: Thank-You After Interview |
| Got an offer, need to evaluate/negotiate | #6: Offer Evaluation & Negotiation |
| Company sent a take-home | #7: Take-Home README |

**Key rule for all templates:** Every message must include either a GitHub repo link or a specific reference to a shipped project. If the template placeholder says `[project]` and you don't have a directly relevant repo, use the closest analogous project and note "same architecture, different domain."
